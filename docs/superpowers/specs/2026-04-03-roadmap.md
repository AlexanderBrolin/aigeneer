## Phase 3: Интерактивные TG-команды (admin chat)

Инженер пишет в чат боту — получает ответ. Примеры:

### Информационные (read-only)
- "какие серверы доступны?" → список серверов из БД с их статусом
- "выполни на autotest2 команду df -h" → SSH → результат в чат
- "покажи логи apache на web-01" → tail error.log → в чат
- "статус репликации на db-01" → SHOW SLAVE STATUS → в чат
- "какие инциденты открыты?" → query DB → список
- "последние проверки" → check_runs за 24ч

### Управляющие (с подтверждением)
- "перезапусти apache на web-01" → кнопка подтверждения → выполнение → результат
- "очисти логи старше 7 дней на web-01" → подтверждение → clear_old_logs
- "перезапусти репликацию на db-01" → подтверждение → restart_replication

### Требования
- Свободная текстовая форма — LLM классифицирует intent
- Read-only команды выполняются сразу, результат в чат
- Write команды — через кнопку подтверждения (interrupt)
- Ответ форматируется: monospace для вывода команд, markdown для текста
- Ограничение по длине: если вывод > 4096 символов — обрезать с пометкой
- Только разрешённые пользователи (TG_ALLOWED_USERS)

### Существующий код
Частично реализовано в `app/agent/graphs/command.py` + `app/bot/handlers.py`.
Нужно: проверить работоспособность, починить, дополнить список серверов/инцидентов.
